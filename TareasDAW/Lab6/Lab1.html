  • ¿Cuáles son las diferencias entre los posibles valores de la propiedad position?

    • Static. El elemento está posicionado de acuerdo al flujo normal de la página. Las propiedades top, right, bottom, left, and z-index no tienen efecto.
    • Relative. El elemento está posicionado de acuerdo al flujo normal, es relativo a sí mismo y es afectado por top, right, bottom, and left
    • absolute. Su posición es fija y está determinada por top, right, bottom, y left. Crea un stackiing context con z-index
    • fixed. El elemento no sigue el flujo normal de la página
    • sticky. se mueve junto con la página

    • inherit
    • initial
    • unset

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • El Internet es la red de redes, es la parte física compuesta por varias computadoras conectadas entre sí. El World Wide Web depende del Internet para trabajar, es gobernado por el Hyper Text Transfer Protocol

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET

      • Solicitud de un recurso. Solo deben recuperar datos

    • HEAD

      • Es la solicitud del header

    • POST

      • Se utiliza para enviar o hacer un registro en un servidor causando a menudo un cambio del estado

    • PUT

      • Reemplaza todo tipo de recurso de destino

    • PATCH

      • Se usa para realizar ediciones o modificaciones parciales a un recurso

    • DELETE

      • Se usa para borrar un recurso determinado

Ciclo de vida de los sistemas de información

  • Planificación. Es el primer paso para el desarrollo de un proyecto. Un a buena planeación es fundamental para el buen desarrollo del proyecto
  • Análisis. En esta etapa se explora e identifica qué es lo que debe hacer el sistema a ser creado. Se llega a una comprensión exacta de cuáles son los requisitos
  • Diseño. Se decide la estructura fundamental del programa, así como la estructura visible para el usuario. Esta estapa es compleja y su realización es iterativa
  • Implementación. Se hace empleo de las herramientas adecuadas y el lenguaje correcto para construir el proyecto
  • Pruebas. Sirve para detectar posibles errores mediante pruebas sistemáticas, para arreglar los problemas antes de la presentación del proyecto
  • Instalación o despliegue. Es identificar y utilizar el software y hardware necesarios para su despliegue. Es donde se implementan los recursos físicos necesarios
  • Uso, manteniemiento. Esta etapa surge una vez que el proyecto ya está completo. Sirve para corregir errores que hayan sido encontrados, así como para adaptar el sistema a nuevas necesidades o funciones

Ciclo de vida de los sistemas de información

  • Delimitación del ámbito del proyecto. Se determina el tamaño del proyecto, aquí se indica todo lo que se necesita hacer
  • Estudio de viabilidad. Se hace el estudio de qué recursos están disponibles y cuáles serán necesarios
  • Análisis de riesgos. Se planifica qué hacer en caso de posibles problemas que surjan durante el desarrollo del proyecto
  • Ciclo de vida clásico. Se conoce como modelo en cascada, se recorre en un orden y se pasa a otra etapa solo cuando se ha finalizado la etapa anterior

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    • El uso regular de !important puede crear desorden en el código y hacer del css algo difícil de mantener. Recomiendo hacer el menor uso posible de éste

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Para evitar que el contenido web como texto se vuelva complicado de leer, además de evitar que la imagen distraiga al usuario de lo que es realmente importante

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    • Depende de qué tan amigable quieras hacer la interfaz a dispositivos con pantallas de diferentes dimensiones. Puedes fijar un tamaño para evitar distorsión o puedes hacer que todo sea relativo al área de visualización. Puedes fijar un tamaño para facilitar la lectura del usuario.

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    • Reduce el tamaño del archivo, se reducen los nombres, se quitan los espacios y utiliza abreviaturas para comprimir las dimensiones. Es una forma optimizada pero difícil de leer para un humano.
